How to Use This Online Keyboard Tester

Follow these four steps to test every key on your keyboard in under a minute.

01
Click on the tester
Click anywhere on the page so your browser focuses this window and captures keyboard events from your physical keyboard.
02
Press every key
Work through each key row by row. Keys turn gold while held and switch to green once detected. Don't forget Fn-keys, numpad, and modifier keys.
03
Find unresponsive keys
Click "Show Untested" to highlight any key that hasn't registered. A key that stays dark after pressing likely has a hardware fault or driver issue.
04
Inspect key codes
The Key Properties panel shows the exact code, key, keyCode, and location for each press — ideal for developers debugging web keyboard input.

If a key doesn't register, try these steps in order: (1) Make sure you've clicked on the page first to focus the browser window. (2) Check if the key registers any response at all — even a partial flicker. (3) Try a different browser. (4) Test the keyboard on another computer. If it fails consistently, the switch or membrane underneath that key is likely faulty.

Note: some keys are intercepted by the OS before reaching the browser — these include the Power key, Fn key, and some media keys. This is normal and not a sign of keyboard damage.

If pressing three or more keys at once causes some not to register, your keyboard has limited rollover (2KRO or 6KRO). Gaming mechanical keyboards with full NKRO can register every key pressed at the same time — all should light up regardless of how many you hold.

code — the physical key identifier regardless of layout (e.g. KeyA, ShiftLeft). This is the modern W3C standard used in web development.

key — the character produced, accounting for modifiers (e.g. A with Shift, a without).

keyCode / which — legacy numeric identifiers still used in older codebases. Both are deprecated but remain in browsers for backwards compatibility.

location — identifies which physical side or group the key is in: Standard, Left, Right, or Numpad. Useful for distinguishing Left Shift from Right Shift.

Yes. The on-screen keyboard shows a standard QWERTY layout, but the key value in the Key Properties panel will reflect whatever character your OS language setting maps to that physical key. So if you're using a French AZERTY layout, pressing the physical Q key will show code: KeyQ but key: a. The code value always refers to the physical key position, not the character output.
